# 记忆功能验证指南

## 启动服务

```bash
cd web-ui
npm run dev        # 启动前端
cd server && npm run dev   # 启动后端（另一个终端）
```

## 验证步骤

### 步骤1：创建记忆

1. 访问 http://localhost:5173/memories
2. 点击"新建记忆"
3. 填写：
   - 标题：`我的技术偏好`
   - 类型：用户信息
   - 内容：`我喜欢用 React + TypeScript 开发项目`
4. 保存

### 步骤2：测试相关记忆（前端）

1. 访问 http://localhost:5173/chat
2. 新建对话
3. 在输入框输入：`我想开发一个前端项目用什么技术好？`
4. **验证点**：输入框上方应出现"相关记忆"面板
   - 显示"我的技术偏好"
   - 显示匹配度如"(85% 匹配)"

### 步骤3：测试记忆自动提取

1. 在聊天中发送：`我喜欢用 Python 做数据分析`
2. 继续聊几句...
3. **验证点**：对话结束后，如果出现绿色提示条：
   - "已自动保存记忆: xxx"
   - 说明服务端自动提取并保存了记忆

### 步骤4：验证记忆文件同步

```bash
# 检查文件是否生成
ls -la .claude/memory/

# 应看到类似文件：
# user_xxxxx.md
```

### 步骤5：验证API端点

```bash
# 获取 token
curl -X POST http://localhost:8080/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"default","password":"default"}'

# 用返回的 token 测试相关记忆API
curl "http://localhost:8080/api/memories/relevant?q=前端技术&limit=3" \
  -H "Authorization: Bearer YOUR_TOKEN"

# 应返回相关记忆列表
```

## 预期行为汇总

| 功能 | 预期表现 |
|------|---------|
| 输入时获取相关记忆 | 输入>10字符后，500ms后显示相关记忆面板 |
| 记忆匹配度 | 显示百分比，如 85% 匹配 |
| 自动提取记忆 | 对话结束后，如检测到有价值信息，显示绿色通知 |
| 记忆注入上下文 | AI回答时会引用记忆中的信息 |
| 文件同步 | .claude/memory/ 目录下生成 .md 文件 |

## 故障排查

- **相关记忆不显示**：检查输入长度是否>10字符，检查网络请求
- **记忆不自动提取**：需要对话至少2轮以上
- **API返回401**：检查token是否过期，重新登录
